# 自動化估值引擎使用說明書 (Valuation Engine Guide)

本系統是一套整合了官方財務數據獲取、結構化清洗、純 Python 數學運算與 AI 語義審計的機構級估值平台。旨在杜絕 AI 在財務分析中的「幻覺」與「敘事偏見」。

## 1. 系統核心理念：數據物理學與邏輯保險絲

本系統不接受無數據支持的成長假設。所有估值必須通過以下審計鏈：

1.  **[精確 EVA 審計]**: 
    *   **強制數據源**: 若需執行精確的 EVA (經濟附加價值) 分析, **必須且僅能** 調用 SEC EDGAR (美股) 或 TEJ (台股) 官方結構化數據。
    *   **禁令**: 嚴禁使用搜尋引擎或第三方新聞數據計算 EVA,以確保模型精度。
2.  **[事實層]**: 僅從 SEC EDGAR (美股) 或 ChinaTimes/TEJ (台股) 獲取經過審計的結構化數據。
2.  **[引擎層]**: 使用純 Python 進行數學運算 (`MetricsEngineDCF_AI`)。
3.  **[讀取層]**: 透過 `AdvancedFinancialReader` 進行趨勢檢測與行業基準對比。
4.  **[保險絲層]**:
    *   **ROIC Gate**: 若 ROIC < WACC，系統會自動否決高成長假設。
    *   **Sustainable Growth**: 成長假設嚴禁超過 `再投資率 * ROIC`。
    *   **Cash Quality**: 若現金流轉化率 < 0.8，估值會自動進行風險折扣。

## 2. 數據缺失與風險補償機制 (Risk Compensation)

在缺乏精確歷史數據時，系統會自動使用以下扣減因子作為風險補償，確保估值不因數據不全而過於樂觀：

| 參數 | 風險觸發條件 | 終端價值 (TV) 影響 |
| :--- | :--- | :--- |
| **Persistence Score** | `StdDev(ROIC - WACC) > 2.5%` | 扣減 15% |
| **ROIC 均值回歸率** | 下滑速度快於 1.0% / 年 | 提升扣減至 20% |
| **數據年限不足** | 歷史數據少於 5 年 | 自動折價 10% |
| **估值倍數 (EV/EBITDA)** | 高於產業平均 20x | 自動折價 10% |

## 3. 核心智能體分工

*   **Orchestrator (Chairman)**: 總控流程，行使邏輯否決權，管理「退化模式」。
*   **Quant Analyst**: 數據與引擎官，負責調度 Python 計算引擎並提供「白話指標」。
*   **Risk Officer**: 審計官，負責 ROIC 均值回歸分析與資本效率審核。
*   **Industry Researcher**: 語義審計員，負責 MD&A 偵測與「敘事偏差」識別。
*   **Lead Valuator**: 情境分析師，產出 Bear/Base/Bull 三種受約束的估值情境。

## 3. 輸出解讀

*   **趨勢標籤 (↑/↓/→)**: 指標的動態變化方向。
*   **白話指標**: 如「產品好不好賺 (毛利率)」、「賺錢效率 (利益率)」。
*   **行業對比**: 自動標註「高於/低於行業平均」。
*   **退化模式標籤**: 若看到 `[DEGRADED MODE]`，代表數據源不全，結果基於 LLM 邏輯推論。
