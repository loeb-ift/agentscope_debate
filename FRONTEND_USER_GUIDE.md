# AI 辯論平台前端操作指南 (Frontend User Guide)

本文件說明如何使用 Gradio 前端介面來管理 Agent、配置辯論以及擴充工具。

**啟動方式**: `docker-compose up --build` 後訪問 `http://localhost:7860`

---

## 1. 👥 Agent 管理 (Agent Management)
在此頁面，您可以創建和管理參與辯論的 AI 角色。

### 操作流程：
1.  **創建 Agent**：
    *   輸入 **名稱** (如 "邏輯大師")。
    *   選擇 **角色** (Debater/Chairman/Analyst)。
    *   填寫 **專長** (如 "經濟學") 和 **系統 Prompt** (定義其性格)。
    *   點擊 **「✨ 創建新 Agent」**。
2.  **編輯/刪除**：
    *   點擊右側列表中的 Agent ID（需手動複製 ID 到左側輸入框）。
    *   點擊 **「✏️ 載入」** 修改後保存，或點擊 **「🗑️ 刪除」**。

---

## 2. ⚔️ 辯論配置 (Debate Configuration)
這是發起辯論的核心頁面。

### 操作流程：
1.  **基礎設定**：輸入 **辯論主題** (如 "AI 是否會取代人類？") 並設定 **輪數**。
2.  **選擇主席**：從下拉選單中選擇一位主席 Agent。
3.  **組建團隊**：
    *   在 **正方團隊** 與 **反方團隊** 下拉選單中，勾選多位 Agent 加入。
4.  **啟動**：
    *   點擊 **「🚀 啟動辯論」**。
    *   右側的 **「📺 實時戰況」** 面板將開始滾動顯示：
        *   主席的賽前分析 (7 步管線)。
        *   Agent 的工具選擇與思考過程。
        *   每一輪的辯論內容與主席總結。
        *   最終的勝負判定與價值昇華。

---

## 3. 🔧 自定義工具 (Custom Tools)
在此頁面，您可以隨時擴充 Agent 的能力，無需重啟系統。

### 新增 HTTP 工具 (如 n8n Webhook)
1.  選擇類型 **`http`**。
2.  輸入 **URL**、**Method** (GET/POST) 與 **Headers**。
3.  填寫 **Schema** (描述參數)。
4.  點擊 **「✨ 自動生成描述」**，讓 AI 幫您寫好給 Agent 看的說明書。
5.  點擊 **「➕ 新增工具」**。

### 新增 Python 工具
1.  選擇類型 **`python`**。
2.  在代碼框中編寫 Python 函數 (必須包含 `def main(**kwargs)` 入口)。
3.  點擊新增，系統會動態編譯並註冊。

---

## 4. 📝 提示詞控制台 (Prompt Console)
進階使用者可在此調整系統的核心指令。

### 操作流程：
1.  選擇一個模板 (如 `chairman.pre_debate_analysis`)。
2.  修改 Prompt 內容 (支持 `{{variable}}` 語法)。
3.  點擊 **「💾 保存」**，下次辯論時將立即生效。
